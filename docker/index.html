Thread

  • Exists within a process
  • Uses process heap
  • Has own stack and register
  • Communicates with other threads by SHM
  • The only thing it isolates is execution(parallelism)

Process

  • Basic system entity
  • Has own memory (heap)
  • Communicates with other processes (ipc, sockets)
  • Adheres to GDT/LDT and kernel-enforced limitations (capabilities, fs permissions)

Container

  • "Virtual home for a process"
  • Shared kernel
  • Netns
  • Capabilities
  • cGroups

More on that later

VM

  • Highest level isolation
  • Hypervisor
  • Emulated architecture
  • GuestOS

More on that later
